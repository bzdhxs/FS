# 对比脚本使用说明

## compare_hho_sghho.py

### 功能
自动运行 HHO 和 SGHHO 两种特征筛选算法，分别配合 PLSR、SVM、RF 三种机器学习模型，共计 6 组实验，并生成对比结果表格。

### 使用方法

```bash
# 在项目根目录下运行
python script/compare_hho_sghho.py
```

### 输出文件

1. **log/comparison_results.md** - Markdown 格式的对比结果表格
   - 包含所有实验的精度指标对比
   - 包含选择的特征数量和特征索引

2. **log/comparison.log** - 详细的运行日志
   - 记录每个实验的执行过程
   - 记录成功/失败状态

3. **log/YYYYMMDD_HHMMSS_<ALGO>_<MODEL>/** - 各实验的详细日志目录
   - 每个实验都会生成独立的日志目录
   - 包含 summary.json、config.json、数据文件、图表等

### 对比指标

脚本会对比以下指标：

- **选择特征数**：算法选择的特征数量
- **训练集R²**：训练集决定系数
- **训练集RMSE**：训练集均方根误差
- **训练集MAE**：训练集平均绝对误差
- **测试集R²**：测试集决定系数
- **测试集RMSE**：测试集均方根误差
- **测试集MAE**：测试集平均绝对误差

### 实验配置

脚本会自动读取以下配置文件：

- **HHO**: `config.yaml`
- **SGHHO**: `config_sghho.yaml`

脚本会自动修改配置文件中的 `model_name` 参数来切换模型。

### 运行时间

根据算法和模型的复杂度，完整运行 6 组实验可能需要：
- **HHO**: 每个模型约 1-5 分钟
- **SGHHO**: 每个模型约 3-10 分钟（因为包含稳定性评估）

总计约 **15-45 分钟**

### 注意事项

1. 确保已安装所有依赖：`pip install -r requirements.txt`
2. 确保数据文件存在：`resource/dataSet.csv`
3. 脚本会自动创建临时配置文件，运行结束后会自动删除
4. 如果某个实验失败，脚本会继续运行其他实验
5. 所有日志都会保存在 `log/` 目录下

### 故障排除

**问题：实验失败**
- 检查 `log/comparison.log` 查看详细错误信息
- 确保配置文件 `config.yaml` 和 `config_sghho.yaml` 存在且格式正确
- 确保数据文件路径正确

**问题：找不到日志目录**
- 检查实验是否真的成功运行
- 查看 `log/` 目录下是否有对应的时间戳目录

**问题：MAE 显示为 N/A**
- 这是因为旧的实验结果中没有 MAE 指标
- 重新运行实验即可获得 MAE 数据
